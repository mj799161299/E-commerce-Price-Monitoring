# 全合规电商价格监控 Web 应用产品详细规划

## 一、产品核心定位

本产品是一款**基于前后端分离架构、完全依托电商平台官方开放 API、全合规的 Web 端价格监控应用**。用户通过浏览器即可访问，无需安装任何软件，支持淘宝 / 天猫、京东、拼多多三大平台，实现商品一键添加、多维度价格阈值设置、定时合规查询、多渠道实时降价提醒。相比桌面端工具，Web 版具备**跨平台无缝访问、多用户隔离、数据云端同步、界面现代化**等核心优势，彻底规避爬虫风险，保障数据获取的长期稳定性与合法性。

## 三、功能需求规划

### （一）用户系统模块

- **用户注册 / 登录**：支持邮箱 / 手机号注册、密码登录，集成验证码防刷机制；
- **个人中心**：用户可修改个人信息、重置密码、查看 API 配额使用情况、管理推送渠道；
- **多用户隔离**：每个用户的监控商品、API 凭证、配置信息完全独立，互不干扰。

### （二）平台 API 凭证管理模块

- **可视化配置界面**：支持用户分平台录入 API 凭证（淘宝联盟 AppKey/AppSecret/pid、京东联盟 AppKey/AppSecret、多多进宝 client_id/client_secret/pid）；
- **一键连通性测试**：实时校验凭证有效性，给出明确错误提示（如凭证错误、权限未开通等）；
- **加密存储**：后端 AES 加密存储用户 API 凭证，杜绝明文泄露风险；
- **配额监控**：实时展示各平台 API 当日调用次数、剩余配额，配额不足时自动推送提醒。

### （三）商品监控管理模块

- **多格式商品添加**：支持输入商品详情页链接、平台短链、淘口令，自动解析提取商品 ID，调用官方 API 获取商品基础信息，一键加入监控列表；
- **监控项全生命周期管理**：支持编辑监控项（修改价格阈值、监控频率）、暂停 / 启用监控、删除监控项、批量操作；
- **监控列表可视化**：表格 / 卡片双视图展示全量监控商品，包含商品缩略图、名称、平台、当前实时价格、设置阈值、降价幅度、监控状态、最后更新时间、直达商品链接。

### （四）价格监控与阈值规则模块

- **多维度阈值设置**：为单个商品独立设置触发规则，支持：

  - 基础阈值：商品实时售价低于 X 元、券后价低于 X 元触发提醒；
  - 幅度阈值：商品价格较上次查询降幅超过 X%、较历史最高价降幅超过 X% 触发提醒；
  - 复合阈值：同时满足「券后价低于 X 元 + 降幅超过 X%」才触发提醒；

  

- **自定义监控频率**：支持全局统一设置 / 单个商品独立设置监控间隔，可选范围 10 分钟 - 24 小时；

- **定时合规查询**：基于定时任务框架，按设定频率调用官方 API 获取最新价格，自动更新数据库，同步校验阈值规则；

- **商品状态自动校验**：API 返回商品下架、不存在、违规封禁时，自动标记监控项为「失效」，并推送提醒告知用户。

### （五）多渠道消息提醒模块

- **多推送渠道集成**：支持 Server 酱、企业微信机器人、钉钉机器人、飞书机器人、邮件等多种提醒渠道，用户可按需配置；
- **触发阈值推送**：推送内容包含商品名称、平台、原价、当前触发价格、设置阈值、降价幅度、商品直达链接；
- **防骚扰设置**：同一商品 24 小时内仅推送 1 次提醒，避免频繁降价导致的消息轰炸。

### （六）前端界面模块

采用现代化响应式设计，适配 PC 端、平板、手机等多设备，核心界面如下：

- **登录 / 注册页**：简洁美观的登录注册界面，支持验证码验证；
- **主界面**：顶部导航栏（个人中心、全局设置、帮助文档）、中部监控列表（表格 / 卡片视图切换）、底部价格趋势预览区；
- **商品添加弹窗**：输入商品链接 / 淘口令，自动拉取商品信息，可视化设置阈值与监控频率；
- **API 配置页**：分平台录入凭证，提供连通性测试、配额查看、官方申请教程跳转入口；
- **历史价格页**：选中商品后，展示历史价格折线图，支持缩放、时间段筛选，直观查看价格波动趋势；
- **个人中心**：用户信息管理、密码修改、推送渠道配置、操作日志查看。

### （七）辅助功能模块

1. **历史价格数据管理**：云端存储全量价格查询记录，支持单商品价格趋势可视化，支持导出历史价格 Excel 文件；
2. **操作日志系统**：记录全量操作行为（API 调用、价格更新、提醒触发、监控启停），支持日志筛选、导出；
3. **数据备份与恢复**：支持一键备份监控列表、历史价格、API 配置，可随时恢复；
4. **帮助文档**：内置 API 申请图文教程、工具使用流程、常见问题排查。

## 五、系统架构设计

采用前后端分离的分层解耦架构，各模块独立封装，便于后续维护与功能扩展，核心架构如下：

### 核心模块职责说明

1. **前端层**：负责用户界面渲染、用户交互处理、状态管理、路由跳转，通过 Axios 调用后端 API，携带 JWT Token 进行鉴权，接收后端返回的 JSON 数据并渲染界面；
2. **后端 API 网关层**：负责接收前端请求，进行 JWT 鉴权验证，路由分发到对应的业务逻辑处理函数，统一处理异常并返回 JSON 格式的响应；
3. **业务逻辑层**：处理核心业务规则，包括用户管理、商品监控管理、阈值规则校验、配额管理、异常处理等，是整个应用的逻辑中枢；
4. **API 统一适配层**：核心中间层，封装三大平台 API 的调用逻辑，屏蔽不同平台的接口差异，对外提供统一的`get_goods_info()`、`parse_goods_id()`方法，返回标准化的商品数据模型；
5. **定时任务模块**：管理全量商品的监控任务，支持单个任务的启停、频率调整，异常任务自动重试；
6. **数据持久层**：通过 SQLAlchemy ORM 操作 PostgreSQL 数据库，存储用户信息、监控商品、历史价格、操作日志等数据，敏感数据全程加密存储。

## 七、开发实施流程

### 阶段 1：需求确认与技术准备（1 周）

1. 完成前后端分离架构的详细设计，确认 API 接口规范、数据格式；
2. 完成三大平台开放 API 的详细调研，确认接口参数、返回格式、权限范围、限流规则；
3. 搭建前后端开发环境，前端集成 Vue 3、Element Plus、Pinia、Vue Router、Vite，后端集成 FastAPI、SQLAlchemy、APScheduler、PyCryptodome、JWT。

### 阶段 2：数据库设计与后端基础模块开发（2 周）

1. 完成数据库表结构设计与初始化，实现敏感数据加密存储逻辑；
2. 开发用户系统模块，实现用户注册、登录、JWT 鉴权、个人中心功能；
3. 开发 API 统一适配层，完成三大平台 API 的封装、商品 ID 解析、标准化数据转换；
4. 开发定时任务模块，实现监控任务的动态添加、启停、频率调整、异常重试。

### 阶段 3：后端业务逻辑与前端界面开发（3-4 周）

1. 开发商品监控管理、阈值规则校验、消息推送、数据可视化等核心业务逻辑，设计并实现前后端 API 接口；
2. 完成前端界面全量页面开发，实现登录 / 注册页、主界面、商品添加弹窗、API 配置页、历史价格页、个人中心，实现界面与后端 API 的联动；
3. 开发数据备份与恢复、日志管理等辅助功能。

### 阶段 4：前后端联调与测试（2 周）

1. 全流程前后端联调测试，验证各功能模块的正确性、API 接口的稳定性；
2. 异常场景测试，处理 API 限流、凭证失效、商品下架、网络异常等边界情况；
3. 性能测试，优化数据库查询、API 响应速度，确保多用户场景下的系统稳定性；
4. 优化界面交互，提升操作流畅度，补充帮助文档与新手引导。

### 阶段 5：部署与发布（1 周）

1. 使用 Docker + Docker Compose 完成容器化部署，配置 Nginx 反向代理前端静态文件、转发后端 API 请求；
2. 编写详细的部署文档、用户使用手册（含 API 申请教程、工具操作流程、常见问题排查）；
3. 发布正式版本，上线运行。

## 八、部署方案

### Docker Compose 部署架构

使用 Docker Compose 一键启动前端、后端、PostgreSQL 数据库三个容器，容器之间通过 Docker 网络通信，配置简单，运维成本低。

#### docker-compose.yml 核心配置示例

yaml

```yaml
version: '3.8'

services:
  db:
    image: postgres:15-alpine
    container_name: price_monitor_db
    environment:
      POSTGRES_USER: price_monitor
      POSTGRES_PASSWORD: your_password
```

### 部署步骤

1. 准备服务器，安装 Docker 和 Docker Compose；
2. 克隆项目代码到服务器，修改 docker-compose.yml 中的数据库密码、JWT 密钥等配置；
3. 进入项目目录，执行`docker-compose up -d`一键启动所有容器；
4. 配置域名解析（可选），通过 Nginx 反向代理实现 HTTPS 访问；
5. 访问服务器 IP 或域名，即可使用 Web 应用。
