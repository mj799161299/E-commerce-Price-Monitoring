## 🚀 新增API接口

### 推送渠道管理
```
POST   /api/push/channels              - 添加推送渠道
GET    /api/push/channels              - 获取推送渠道列表
PUT    /api/push/channels/{id}         - 更新推送渠道
DELETE /api/push/channels/{id}         - 删除推送渠道
POST   /api/push/channels/{id}/test    - 测试推送渠道
```

## 📝 使用示例

### 1. 配置Server酱推送

```bash
# 请求
POST /api/push/channels
Authorization: Bearer <token>

{
  "channel_type": "serverchan",
  "channel_config": {
    "sendkey": "your_sendkey_here"
  }
}

# 响应
{
  "id": 1,
  "channel_type": "serverchan",
  "is_active": true
}
```

### 2. 测试推送

```bash
# 请求
POST /api/push/channels/1/test
Authorization: Bearer <token>

# 响应
{
  "success": true,
  "message": "测试推送发送成功"
}
```

### 3. 添加监控商品（自动创建定时任务）

```bash
# 请求
POST /api/monitor/items
Authorization: Bearer <token>

{
  "goods_url": "https://item.taobao.com/item.htm?id=123456",
  "threshold_price": 99.99,
  "threshold_type": "price",
  "monitor_interval": 60
}

# 系统会自动：
# 1. 解析商品ID
# 2. 获取商品信息
# 3. 创建监控记录
# 4. 添加定时任务（每60分钟检查一次）
```

## 🔧 系统工作流程

### 价格监控流程

```
1. 用户添加商品
   ↓
2. 系统创建定时任务
   ↓
3. 定时任务按设定频率执行
   ↓
4. 调用平台API获取最新价格
   ↓
5. 记录价格历史
   ↓
6. 检查是否触发阈值
   ↓
7. 触发阈值 → 发送推送通知
```

### 推送通知流程

```
1. 检测到价格触发阈值
   ↓
2. 查询用户配置的推送渠道
   ↓
3. 遍历所有启用的渠道
   ↓
4. 解密推送配置
   ↓
5. 调用对应推送服务
   ↓
6. 发送降价提醒
```

## ⚠️ 注意事项

### 1. 定时任务

- 定时任务在应用启动时自动启动
- 添加商品时会自动创建对应的定时任务
- 删除商品时会自动移除定时任务
- 暂停商品不会移除任务，只是跳过执行

### 2. 推送配置

- 推送配置使用AES加密存储
- Server酱需要申请sendkey: https://sct.ftqq.com/
- 企业微信/钉钉/飞书需要创建机器人获取webhook
- 邮件推送需要SMTP服务器配置

### 3. API调用

- 目前使用模拟数据，实际需要对接真实平台API
- 需要申请淘宝联盟、京东联盟、多多进宝的API凭证
- 注意API调用频率限制
